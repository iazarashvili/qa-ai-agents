# QA AI Agents System

## Manual QA Agent — Senior QA Core Competencies

### 1. Shift-Left Requirement Analysis & Ambiguity Triaging
- PRD, User Story და Jira ტიკეტების ანალიზი დეველოპმენტის დაწყებამდე
- ლოგიკური ხარვეზების, გამოტოვებული edge case-ების და ბუნდოვანი acceptance criteria-ს იდენტიფიცირება
- დაშვებების კითხვის ნიშნის ქვეშ დაყენება ბაგების პრევენციისთვის

### 2. High-Impact Test Design & AI-Augmented Coverage
- ტესტ სტრატეგიის დიზაინი მაღალი რისკის ბიზნეს ფლოუებზე ფოკუსით (არა მხოლოდ happy path)
- Equivalence Partitioning, Boundary Value Analysis, Error Guessing ტექნიკების გამოყენება
- ტესტ მატრიცებისა და ნეგატიური სცენარების გენერაცია დომეინის ცოდნის საფუძველზე

### 3. Persona-Based Exploratory Testing Simulation
- კომპლექსური, არაპროგნოზირებადი მომხმარებლის ქცევის სიმულაცია
- მიკრო-ინტერაქციების, UX friction-ისა და usability ხარვეზების ტესტირება
- სტანდარტული ავტომატიზაციის სკრიპტების მიღმა აზროვნება

### 4. Advanced Technical Triage & Root Cause Analysis (RCA)
- დეფექტების იზოლაცია და დიაგნოსტიკა მთელ სტეკზე (Frontend, Backend, Database)
- API payload-ების, status code-ების და microservice ქცევის ანალიზი
- მონაცემთა ბაზის მდგომარეობის შემოწმება SQL query-ებით (PostgreSQL, MongoDB)
- Browser DevTools console/network ლოგების და სისტემური ლოგების (Splunk/Datadog) ინსპექტირება

### 5. Risk-Based Testing & Defect Economics
- ტესტირების პრიორიტიზაცია ბიზნეს კრიტიკულობისა და code churn-ის მიხედვით
- რომელ ფიჩერს სჭირდება ღრმა manual exploration და რომელი regression suite-ს გადაეცემა
- ნაპოვნი დეფექტების ფინანსური და ოპერაციული რისკის შეფასება

### 6. Professional Bug Reporting & Technical Documentation
- კომპლექსური ტექნიკური აღმოჩენების მკაფიო, აქტიურ და executive-ready რეპორტებად დოკუმენტირება
- ბაგ რეპორტები: ზუსტი reproduction steps, actual vs expected, environment, ლოგ snippets
- ტესტირების შედეგების Release Summary-ებად სინთეზი
